UI Automation

Building robust Selenium/Playwright frameworks with Page Object Model design pattern.

public LoginPage extends BasePage {
  WebElement username = $("#username");
  WebElement password = $("#password");
  WebElement submit = $("#submit");

  public void login(String user, String pass) {
    username.sendKeys(user);
    password.sendKeys(pass);
    submit.click();
  }
}

QA War Stories

The Password Reset Bug

Discovered that password reset functionality was sending plain-text passwords via email instead of reset links. This security vulnerability affected 50,000+ users before launch.

Lesson: Always test security-critical flows end-to-end, not just happy paths.

The Currency Conversion Edge Case

Found that currency conversion was failing for amounts over 1 trillion due to integer overflow. The bug was hidden because "who sends a trillion dollars?" – except institutional clients do.

Lesson: Test extreme edge cases, even if they seem unrealistic.

The Mobile Memory Leak

Identified a memory leak in the mobile app that only manifested after 30+ minutes of continuous use. Performance testing revealed the app would crash for power users.

Lesson: Performance testing is as important as functional testing.
